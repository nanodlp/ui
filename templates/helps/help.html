Troubleshooting

Image Issues

No Picture

  • Raspberry Pi: Wrong HDMI mode, edit config.txt file.
  • Desktop Version: Display is not started from dashboard.
  • Wrong display no.
  • Black or color close to black has been chosen as layer rendering color.


Image is flipped

  • Raspberry Pi: edit config.txt file.
    • 90 degrees - display_rotate=1
    • 180 degrees - display_rotate=2
    • 270 degrees - display_rotate=3
    • horizontal flip - display_rotate=0x10000
    • vertical flip - display_rotate=0x20000
  • Projector: You can correct image flipping issue using projector options.


Print Issues

Print dimension issue on X/Y

  • Projector resolution on setup page is wrong.
  • X/Y resolution on setup page is wrong.
  • Override X/Y Resolution on profile page is wrong.
  • Modify x/y resolution on profile page is wrong.


Movement Issues

Movement distance is wrong

  • RAMPS: Wrong positioning mode from the setup page (Relative/Absolute)
  • Direct Control: Z-Axis details are wrong on the setup page.
  • Direct Control: Movement is too fast, steps get skipped. Change movement speed.

Movement direction is wrong

  • Change axis direction from the setup page


Slicer Issues

Missing or additional parts on a layer

  • Source STL file has issue, share it so we could analyse if we could handle the problem or not.


NanoDLP Issues

Program crashes or reboot

  • Go to setup page / tools tab and press debug button to save log files belong to the crash, and share it with us.
  • In some cases crashes are related to power supply issues. Make sure the power supply is reliable and powerful enough.
